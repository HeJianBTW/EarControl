# EarControl 免费测试版安装指南

EarControl 当前通过 GitHub 免费开源测试，不经过 Mac App Store，也尚未使用 Developer ID 公证。macOS 会要求你手动确认首次打开和辅助功能权限。

## 安装

1. 从 GitHub Releases 下载 `EarControl-v1.7.5-macOS.dmg`。
2. 打开 DMG，把 `EarControl.app` 拖到旁边的 `Applications` 文件夹。
3. 从“应用程序”打开 EarControl。
4. 如果 macOS 阻止打开：
   - 打开“系统设置 → 隐私与安全性”；
   - 向下找到 EarControl 被阻止的提示；
   - 点击“仍要打开”，再次确认。
5. EarControl 会自动显示首次设置检查。

## 权限

在首次设置窗口中完成以下项目：

1. 插入 Apple 有线 EarPods。
2. 点击“打开设置…”，在“隐私与安全性 → 辅助功能”中打开 EarControl。
3. 完全退出并重新打开 EarControl，让新授权进入应用进程。
4. 切回 EarControl。应用会重新检查并重建监听。
5. 只有窗口显示“当前连接模式需要输入监控”时，才开启“输入监控”。
6. 在“系统设置 → 声音 → 输入”中选择 `External Microphone`，如果你希望使用耳麦收音。

辅助功能授权只能由当前 Mac 的用户亲自开启，不能随 DMG 预装。

## 微信输入法

在“微信输入法 → 语音输入”中，把“按住说话”设置为右 Option。EarControl 默认也使用右 Option。

## 实际验证

依次按下三枚线控键，首次设置窗口应显示 `3/3`。这一步只验证硬件信号；还要确认“辅助功能身份”和“键盘事件发送”都显示通过：

- 音量减：开始并保持语音输入；
- 中间键短按：结束语音并等待整理；
- 中间键单击、双击、三击：执行各自在设置中选择的快捷键或预设动作；
- 中间键长按：结束语音并清空输入框；
- 音量加：整理后发送。

完整完成一次“开始说话 → 结束 → 发送”后，“语音工作流”会显示已通过。

## 更新后权限失效

免费测试版使用稳定的 Apple Development 测试签名，但尚未经过 Developer ID 公证。升级到签名不同的旧版本或自行临时签名构建的版本后，macOS 可能把它识别为新的代码身份。若按钮会亮但没有实际按键动作：

1. 打开 EarControl 的“设置检查”；
2. 查看“辅助功能身份”和“键盘事件发送”；
3. 先完全退出并重新打开 EarControl；
4. 如果仍未生效，再前往“系统设置 → 隐私与安全性 → 辅助功能”；
5. 删除旧 EarControl 项目，重新添加 `/Applications/EarControl.app` 并打开；不要添加 DMG、下载目录或旧备份里的副本；
6. 再次完全退出并重新打开 EarControl。

## 诊断信息

打开“EarControl → 设置 → 按键诊断”，点击“复制诊断信息”。诊断只包含版本、权限布尔状态、HID 事件数量、当前输入设备名称和最近映射动作，不包含录音或输入文字。
