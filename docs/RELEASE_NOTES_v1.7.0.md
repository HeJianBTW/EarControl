# EarControl v1.7.0

这是 EarControl 的免费开源测试版，重点解决“界面能够识别线控，但 macOS 实际拦截映射动作”时难以判断原因的问题。

## 新增

- 首次启动设置检查，逐项显示 HID 连接、辅助功能身份、键盘事件发送、输入监控和当前麦克风。
- 三键原始事件验证，确认音量加、中间键和音量减均被识别。
- 完整语音工作流验证，确认“开始说话 → 结束 → 发送”已经执行。
- 权限从关闭变为开启后自动刷新状态并重建监听。
- 设置中的可复制诊断信息，不包含录音或输入文字。
- 免费测试 DMG、安装说明和 GitHub 安装文档。

## 保持不变

- 音量减：开始并保持右 Option 语音输入。
- 中间键短按：结束语音并等待整理。
- 中间键长按：结束语音并清空输入框。
- 音量加：整理后发送。
- HID 独占拦截，避免改变系统音量或启动 Apple Music。

## 安装提醒

该版本使用临时签名，尚未经过 Developer ID 公证。首次打开需要在“系统设置 → 隐私与安全性”中选择“仍要打开”，并亲自开启辅助功能权限。升级后如果系统保留了旧的同名授权但映射不执行，请删除旧 EarControl 权限条目，再重新添加 `/Applications/EarControl.app`。

完整步骤见仓库中的 `docs/INSTALL.md`。
